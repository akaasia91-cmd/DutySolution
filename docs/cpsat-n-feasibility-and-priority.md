# N 근무 규칙·우선순위 분석 메모 (E1 10명·일 N 2명 전제)

대화에서 정리한 **생각의 단계**를 보존한다. 구현 변경 여부와 무관하게 참고용.

## 1단계: 논리 분석

- **전제:** 총원 10명 → 통상 수간(0) + 일반 9명이 N 후보. 월 30일 × 매일 N 2명 = **60슬롯**. 인당 월 N 상한 7이면 이론 상한 `9×7=63 ≥ 60` → 개수만 보면 여유.
- **2~3연속 블록 + 월 최대 7:** 블록을 모두 3일로만 채우면 `9>7`로 불가. 실제로는 2·3의 조 합이 7이 되게 쪼개야 함. **단독 1N 금지(말일 예외)**면 더 빡빡함.
- **블록 간 최소 5일(비N):** 블록 끝 다음날부터 다음 블록 시작 전까지 비N가 최소 5일이면, 한 사람 타임라인이 빠르게 소모됨. **9명이 서로 위상을 어긋나게** 깔지 않으면 특정일 N 가능 인원이 2명 미만이 될 수 있음.
- **간격 최대 7일**(규칙에 포함될 때): 민·맥 동시에 있으면 다음 N 시기가 거의 고정에 가까워져 **회전 설계 난이도 증가**.
- **결론:** “항상 가능”이라 단정할 수 없음. 고정 신청·휴가·임산부·이월 등에 따라 **INFEASIBLE**이 나올 수 있는 영역.

## 2단계: 우선순위(의도) vs 구현 시 유의점

**의도된 순위(요청 기준)**

1. 매일 N 2명 강제  
2. N는 2~3 연속 블록, 단독 1블록 금지(말일 예외)  
3. N–N 간격 최소 5일  

**구현과 어긋날 수 있는 점:** 일별 `need_n`이 하드가 아니라 슬랙+목적 가중인 반면, N 패턴·직후 휴무·간격 등이 하드이면, INFEASIBLE 시 **“패턴을 먼저 지키려다 일일 2N을 못 채움”**으로 읽히는 구조가 될 수 있음. 의도와 맞추려면 하드/소프트 재배치 또는 단계적 최적화 검토가 필요할 수 있음.

## 3단계: 가상 시뮬레이션(직관)

- 한 사람이 `2N–5비N–2N–…` 패턴이면 한 달에 블록 2~3번·7 상한 근접.  
- 9명이 같은 위상이면 같은 날 N 불가가 겹칠 수 있음 → **일부러 시작일·블록 길이를 분산**해야 매일 2명 유지 가능성이 생김.  
- 말일 단독 N 예외 사용자는 다른 이들과 맞물림이 달라져 **솔버/검증으로 함께 보는 것**이 현실적.
